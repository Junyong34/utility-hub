# 02차: 웹사이트 구조 및 컬러코드 디자인 환경 구성

> 상세화 전 검토 체크포인트  
> - 사이트의 정보구조(IA)와 핵심 페이지군이 정의되었는가?  
> - 컬러/타이포/간격 규칙이 토큰 단위로 관리 가능한가?  
> - 디자인 기준이 구현 세부사항보다 상위 원칙 중심으로 정리되었는가?

## 목적

- 사이트 전체 구조(정보 흐름)를 고정한다.
- 색상/타이포/스페이싱 기준을 디자인 토큰 관점으로 정의한다.
- 이후 페이지 설계 단계의 일관성을 보장한다.

## 범위(포함/제외)

- 포함:
  - 상위 네비게이션과 페이지군 관계 정의
  - 컬러 토큰 체계(기본/보조/상태색) 정의
  - 타이포/간격 스케일 원칙 정의
- 제외:
  - 상세 시안 제작
  - 페이지별 픽셀 단위 레이아웃 고정

## 선행조건

- 01차 기술 기준이 확정되어 있어야 한다.
- 브랜드 톤(전문성/신뢰/가독성) 방향 합의가 필요하다.

## 핵심 작업

- IA 드래프트 작성(홈, 블로그, 유틸 허브, 상세 페이지군)
- 디자인 토큰 네이밍 규칙 정의
- 접근성 고려 대비 대비비/가독성 기준 정의
- 공통 레이아웃 블록(헤더/본문/푸터) 역할 정의

## 산출물

- IA 구조도(텍스트 또는 표)
- 컬러/타이포/간격 토큰 초안
- 품질 기준(가독성/일관성/확장성) 체크리스트

## 의사결정 포인트

- 색상 체계의 기준 수(최소 팔레트 vs 확장 팔레트)
- 모바일 우선 설계 여부
- 레이아웃 폭/콘텐츠 밀도 기본값

## 리스크

- 토큰 기준이 약하면 페이지별 디자인 편차 증가
- 대비비 미준수 시 접근성/SEO 품질 저하 가능

## 완료 기준

- IA와 페이지군 관계가 문서로 확정됨
- 토큰 규칙이 다음 단계 구현에 바로 전달 가능함
- 품질 기준 체크리스트가 명시됨

## 다음 단계 인계

- 광고 슬롯 위치 논의 시 참조할 레이아웃 블록 정의
- 라우팅 전략 수립 시 사용할 페이지군 분류 기준

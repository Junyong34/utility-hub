# 05차: 라우터별 페이지 상세 설계 및 콘텐츠 자산 준비

> 상세화 전 검토 체크포인트  
> - 라우터별 템플릿이 구현/콘텐츠/SEO 관점을 모두 담고 있는가?  
> - 글 대본 구조와 이미지 캡처 기준이 재사용 가능하게 정리되었는가?  
> - 이번 단계가 "상세 설계 준비"인지 "실제 제작"인지 경계가 분명한가?

## 목적

- 04차 라우팅 정의를 기반으로 페이지 설계 템플릿을 만든다.
- 블로그/유틸 페이지 공통으로 사용할 콘텐츠 스크립트 구조를 정의한다.
- 이미지 캡처 준비 기준을 문서화해 제작 단계 품질 편차를 줄인다.

## 범위(포함/제외)

- 포함:
  - 라우트별 페이지 설계 템플릿
  - 글 대본(문제-해결-요약) 구조
  - 이미지 캡처 체크리스트
- 제외:
  - 실제 원고 작성 완료
  - 실제 이미지 제작/편집 완료

## 선행조건

- 04차 라우트 매트릭스 확정이 필요하다.
- 02차 디자인 시스템 기준이 준비되어야 한다.

## 핵심 작업

- 페이지 설계 템플릿 항목 고정(헤드라인/핵심 섹션/CTA)
- 콘텐츠 타입별 대본 구조 정의(정보형/도구형)
- 스크린샷 기준 정의(해상도/뷰포트/노출 요소)

## 산출물

- 라우트별 상세 설계 템플릿 문서
- 콘텐츠 대본 구조 가이드
- 이미지 캡처 준비 체크리스트

## 의사결정 포인트

- 페이지 템플릿을 단일화할지 유형별로 분리할지
- 이미지 캡처 자동화 도입 시점
- 콘텐츠 승인 흐름(작성-검토-게시) 단순화 수준

## 리스크

- 템플릿 누락 시 페이지 품질 편차 증가
- 캡처 기준 부재 시 블로그/도구 화면 일관성 저하

## 완료 기준

- 주요 라우트에 대응되는 템플릿이 모두 존재함
- 대본 구조와 캡처 체크리스트가 문서화됨
- 상세 제작 전에 필요한 입력값이 확정됨

- 07~08차 광고 단계에서 사용할 콘텐츠 블록 위치 정보 전달

# 06차: SEO 최적화 작업 및 검색엔진 등록

> 상세화 전 검토 체크포인트  
> - 메타/구조화데이터/사이트맵 전략이 라우트 구조와 정합한가?  
> - 검색엔진 등록 절차에서 필요한 소유권 검증 정보가 준비되었는가?  
> - 기술 SEO와 콘텐츠 SEO의 역할 분리가 명확한가?

## 목적

- 검색엔진 노출을 위한 기술 SEO 기본 체계를 확립한다.
- 검색엔진 등록 절차를 단계적으로 정리한다.
- 이후 운영 단계에서 반복 가능한 SEO 점검 기준을 만든다.

## 범위(포함/제외)

- 포함:
  - 메타 태그/OG/robots/sitemap/구조화 데이터 작업 골격
  - 검색엔진(예: Google, Naver) 등록 절차 골격
  - SEO 품질 점검 항목 정의
- 제외:
  - 키워드 리서치 심화
  - 대규모 콘텐츠 리라이팅 실행

## 선행조건

- 04차 라우트 정보와 05차 템플릿 정보가 필요하다.
- 도메인/배포 URL이 준비되어야 검색 등록 절차 수행이 가능하다.

## 핵심 작업

- 라우트별 메타 기본 규칙 정의
- 사이트맵/robots 관리 기준 정의
- 구조화 데이터 적용 대상 식별
- 검색 콘솔 등록 체크리스트 작성

## 산출물

- SEO 작업 체크리스트
- 검색엔진 등록 체크리스트
- 운영 점검 주기(주간/월간) 초안

## 의사결정 포인트

- 메타 자동 생성 범위
- 구조화 데이터 타입 우선순위(Article, FAQ, WebPage 등)
- 검색엔진별 등록 우선순위

## 리스크

- 메타 정책 누락 시 인덱싱 품질 저하 가능
- 검색엔진 정책/도구 변경으로 절차 수정 필요 가능

## 완료 기준

- 기술 SEO 작업 목록이 라우트 기준으로 정리됨
- 검색 등록 절차가 재실행 가능한 체크리스트로 정리됨
- 운영 점검 항목과 책임 범위가 문서화됨

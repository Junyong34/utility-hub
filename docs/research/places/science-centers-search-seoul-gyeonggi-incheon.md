## import 메모

- 최종 문서는 `placeId` 기준으로만 중복 제거했습니다. 같은 `placeId`가 여러 번 노출된 경우는 하나로 합쳤습니다.
- 경기권은 검색 및 상세 검증 기준으로 0건이라, 섹션에 0건 문구만 남겼습니다.
- 테이블의 `상세 API 메모`는 API 응답에서 확인된 `createdAt`, `updatedAt`, `hiddenKeywords`, `tip`, `reserveDescription`, `amenities`, `usageStat`를 바탕으로 `/places` import 적합성을 짧게 적은 해석입니다.
- `meta.updatedAt`은 별도 메타 엔드포인트의 갱신 시점이므로, 본문 수정 시점과 분리해서 보면 됩니다.
- `mainKeyword`는 전부 `과학관`이었고, `분류`는 모두 `박물관/체험관`이었습니다.

## 결론

서울과 인천에 과학관 성격의 장소가 집중되어 있었고, `/places` import용으로는 예약 여부, 운영시간, 편의시설, 연령 신호를 함께 붙이기 좋은 구성이었습니다. 경기권은 이번 검색 조건에서는 0건이었습니다.

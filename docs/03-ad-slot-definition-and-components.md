# 03차: 웹사이트 광고 삽입 위치 정의 및 컴포넌트 개발

> 상세화 전 검토 체크포인트  
> - 광고 위치 결정이 UX/성능/정책 충돌 없이 가능한가?  
> - 광고 렌더링 로직을 공통 컴포넌트로 분리할 기준이 있는가?  
> - 네트워크별 정책 차이를 반영할 유연성이 확보되는가?

## 목적

- 광고 삽입 후보 위치를 페이지 유형별로 정의한다.
- 광고 관련 UI/로직을 재사용 가능한 컴포넌트 단위로 분리하는 기준을 수립한다.
- 정책 검토와 롤아웃 단계를 위한 슬롯 인벤토리 기반 문서 틀을 만든다.

## 범위(포함/제외)

- 포함:
  - 페이지 타입별 광고 슬롯 후보 정의
  - 광고 컴포넌트 책임 범위(표시/로드/폴백) 정의
  - 정책 리스크 체크 항목 정의
- 제외:
  - 실제 광고 스크립트 삽입
  - 광고 수익 최적화 A/B 테스트

## 선행조건

- 02차 레이아웃 블록 정의가 필요하다.
- 광고 네트워크 정책 확인을 위한 계정 준비가 필요할 수 있다.

## 핵심 작업

- 광고 슬롯 네이밍 규칙 수립
- 페이지 유형별 기본 위치 후보 정의
- 광고 컴포넌트 인터페이스(입력값/폴백 전략) 초안 정의
- 정책 위반 가능 영역 사전 체크

## 산출물

- 광고 슬롯 인벤토리 표(초안)
- 광고 컴포넌트 역할 분리 원칙 문서
- 정책 리스크 체크리스트

## 의사결정 포인트

- 슬롯 수를 최소화할지(보수적) 확장할지(공격적)
- Above-the-fold 광고 허용 범위
- 광고 로드 시점(초기 로드 vs 지연 로드)

## 리스크

- 광고 밀도 과다 시 UX 하락 및 정책 이슈 가능
- 네트워크 정책 변화에 따른 재작업 발생 가능

## 완료 기준

- 광고 슬롯 인벤토리가 페이지 유형별로 정의됨
- 컴포넌트 역할/경계가 문서로 확정됨
- 정책 리스크 항목이 점검 가능한 형태로 정리됨

## 다음 단계 인계

- 04차 라우팅 설계 시 슬롯 배치 가능 페이지군 정보 전달
- 07~08차 네트워크 준비/삽입 단계에서 재사용할 슬롯 ID 세트 전달

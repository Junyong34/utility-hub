# AI 기반 통계 추천 알고리즘

## 개요

본 문서는 로또 번호 추천 서비스에서 사용하는 **AI 기반 통계 추천 알고리즘**의 상세 흐름과 가중치 계산 공식을 정의합니다.

> ⚠️ **중요**: 이 알고리즘은 엔터테인먼트 목적으로 설계되었으며, 실제 당첨 확률을 높이지 않습니다. 로또는 완전 무작위 추첨이며, 모든 번호 조합의 당첨 확률은 동일합니다.

## 가중치 계산 공식

### 전체 공식

```
번호별 최종 가중치 W(n) =
    (전체빈도율 × 0.40)
  + (최근100회차빈도율 × 0.35)
  + (동반출현보정값 × 0.15)
  + (균형보정값 × 0.10)
```

### 1. 전체 빈도 가중치 (40%)

**의미**: 1~1,000회차 전체에서 번호 n이 출현한 빈도를 정규화한 값

**계산식**:
```
F_weight(n) = (count(n) / max_count) × 0.40

여기서:
- count(n): 번호 n의 전체 출현 횟수
- max_count: 전체 번호 중 최대 출현 횟수
```

**예시**:
- 번호 34: 120회 출현 (최대 출현 번호)
- 번호 27: 110회 출현
- F_weight(34) = (120 / 120) × 0.40 = 0.40
- F_weight(27) = (110 / 120) × 0.40 = 0.367

### 2. 최근 빈도 가중치 (35%)

**의미**: 최근 100회차 내에서 번호 n이 출현한 빈도 (트렌드 반영)

**계산식**:
```
R_weight(n) = (recent_count(n) / max_recent_count) × 0.35

여기서:
- recent_count(n): 최근 100회차 내 번호 n 출현 횟수
- max_recent_count: 최근 100회차 내 최대 출현 횟수
```

**예시**:
- 번호 13: 최근 100회차 중 16회 출현 (최대)
- 번호 45: 최근 100회차 중 14회 출현
- R_weight(13) = (16 / 16) × 0.35 = 0.35
- R_weight(45) = (14 / 16) × 0.35 = 0.306

### 3. 동반 출현 가중치 (15%)

**의미**: 다른 번호들과 함께 당첨된 빈도를 기반으로 한 상관관계 점수

**계산 방법**:
1. 모든 번호 쌍(pair)의 동반 출현 횟수를 집계
2. 각 번호의 동반 출현 점수 = 해당 번호가 포함된 모든 pair의 출현 횟수 합계
3. 정규화 (0~1 범위)

**계산식**:
```
C_score(n) = Σ co_occurrence(n, m) / max_score

여기서:
- co_occurrence(n, m): 번호 n과 m이 함께 당첨된 횟수
- m: 1~45의 모든 다른 번호
- max_score: 모든 번호 중 최대 동반 출현 점수

C_weight(n) = C_score(n) × 0.15
```

**예시**:
- 번호 34: 다른 번호들과 총 520회 동반 출현
- 번호 27: 다른 번호들과 총 495회 동반 출현
- 최대 동반 점수: 520회
- C_weight(34) = (520 / 520) × 0.15 = 0.15
- C_weight(27) = (495 / 520) × 0.15 = 0.143

### 4. 균형 보정 가중치 (10%)

**의미**: 홀짝 균형과 구간 분산을 위한 보정값

**계산식**:
```
B_weight(n) = (odd_bonus + range_bonus)

여기서:
- odd_bonus = 0.5 (모든 번호 동일)
- range_bonus = 0.3 (1~10, 41~45 구간)
- range_bonus = 0.5 (11~40 중간 구간)
```

**예시**:
- 번호 5 (1~10 구간): B_weight = (0.5 + 0.3) = 0.8 × 0.10 = 0.08
- 번호 25 (중간 구간): B_weight = (0.5 + 0.5) = 1.0 × 0.10 = 0.10
- 번호 43 (41~45 구간): B_weight = (0.5 + 0.3) = 0.8 × 0.10 = 0.08

## 가중치 기반 샘플링 알고리즘

**Weighted Random Sampling** 방식 사용:

```typescript
1. 모든 번호(1~45)의 가중치 합계를 계산: total_weight
2. 0~total_weight 범위에서 랜덤 값 r 생성
3. 번호를 순회하면서 가중치를 누적:
   - r -= weight(n)
   - r <= 0이면 번호 n을 선택
4. 선택된 번호는 제외하고 1~3을 반복 (총 6회)
```

**확률 예시**:
- 번호 34 (가중치 0.95) 선택 확률: 0.95 / total_weight
- 번호 2 (가중치 0.42) 선택 확률: 0.42 / total_weight

## 유효성 검증 기준

### 1. 홀짝 비율 (권장)
- **조건**: 홀수 2~4개, 짝수 2~4개
- **예시**: 3:3, 2:4, 4:2 ✓ / 1:5, 6:0 ✗

### 2. 구간 편중 방지
- **조건**: 단일 구간(1~10, 11~20 등)에 4개 이상 집중 방지
- **예시**: [1, 3, 5, 7, 9, 11] (1~10 구간 5개) ✗

### 3. 연속번호 제한
- **조건**: 3개 이상 연속 번호 방지
- **예시**: [1, 2, 3, 15, 20, 30] (1,2,3 연속) ✗

### 4. 합계 범위 (선택적)
- **권장 범위**: 100~200
- **통계적 평균**: 약 135~175

## 성능 최적화

### 캐싱 전략
- 전체 통계 분석 결과: 24시간 캐시
- 최근 100회차 통계: 1시간 캐시
- 가중치 테이블: 동행복권 추첨 시 업데이트 (주 1회)

### 재시도 정책
- 최대 재시도: 50회
- 재시도 후에도 검증 실패 시: 순수 랜덤 번호로 폴백

## 알고리즘 한계 및 투명성

### 한계
1. **당첨 확률 향상 불가**: 로또는 완전 무작위 추첨으로, 과거 데이터는 미래 당첨과 무관합니다.
2. **패턴의 허상**: 통계적 패턴은 우연의 일치일 가능성이 높습니다.
3. **가중치의 자의성**: 가중치 비율(40%, 35%, 15%, 10%)은 경험적 설정이며 과학적 근거는 없습니다.

### 투명성 원칙
- 본 알고리즘은 사용자에게 "재미있는 번호 선택 경험"을 제공하기 위한 것입니다.
- 실제로는 가중치를 적용한 랜덤 생성이며, AI나 머신러닝 모델을 사용하지 않습니다.
- 사용자에게 당첨을 보장하거나 확률을 높인다는 표현을 사용하지 않습니다.

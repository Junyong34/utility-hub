# 비 오는 날 실내 콘텐츠 클러스터 설계

## 요약

이 문서는 육아 정보 웹사이트의 검색 유입을 늘리기 위한 4주짜리 SEO 콘텐츠 클러스터 설계다.

승인된 방향:

- 트래픽 목표: 빠른 검색 유입 증가
- 콘텐츠 축: 상황형 콘텐츠
- 첫 클러스터: 비 오는 날과 실내 나들이
- 검색 채널: 구글식 구조와 한국어 생활 검색어 감각을 함께 반영
- 발행 속도: 4주 동안 주 2개, 총 8개

이 클러스터는 얕은 대량 리스트 글을 만들기 위한 것이 아니다. 기존 `content/places` 장소 seed를 사실 정보의 기준으로 삼고, 각 글에서는 부모가 실제로 고를 때 필요한 판단 맥락을 더한다.

## 현재 맥락

사이트는 서울·경기·인천의 1~7세 자녀 부모를 위한 실용형 육아 가이드로 재정의되고 있다.

이미 있는 자산:

- `/places`와 지역별 장소 페이지가 존재한다.
- `content/places`에 서울, 경기 남부, 경기 북부, 인천 장소 seed가 있다.
- 홈과 장소 UI에 실내, 우천 가능, 무료, 연령 필터가 이미 노출된다.
- 현재 블로그 글은 `parking`, `benefits`에 비해 상황형 육아 검색 콘텐츠가 부족하다.

설계 시점의 장소 seed 수:

- 서울: 98
- 경기 남부: 58
- 경기 북부: 35
- 인천: 33
- 합계: 224

## 목표

비 오는 날 아이와 바로 갈 수 있는 실내 장소를 찾는 부모의 검색 유입을 늘린다.

우선 포착할 검색 의도:

- `비오는날 아이와 가볼만한곳`
- `비 오는 날 아이랑`
- `아이와 실내 가볼만한곳`
- `서울 아이랑 실내`
- `무료 실내 아이와`
- `예약없이 아이와 갈만한곳`
- `아기랑 실내 갈만한곳`
- `유아 실내 체험`

## 비목표

- 이번 단계에서 전국 단위 장소 디렉토리를 만들지 않는다.
- `/places?rain=true`, `/places?indoor=true` 같은 필터 URL을 주 색인 대상으로 밀지 않는다.
- 같은 장소 설명을 여러 글에 복붙하지 않는다.
- `discovery_only`, `needs_refresh` 장소를 대표 추천에 넣지 않는다.
- 기존 주차, 금융, 소비자, 혜택, 도구 콘텐츠를 삭제하지 않는다.

## 추천 접근

상황형 피라미드를 사용한다.

넓은 검색 의도를 받는 허브 글 1개를 만들고, 하위 글 7개가 지역, 비용, 당일성, 연령 검색 의도를 나눠 받는다. 허브 글은 모든 하위 글로 연결하고, 하위 글은 허브 글과 관련 지역/조건 글로 되돌아간다.

이 접근이 지역형 또는 장소유형형보다 적합한 이유는 부모의 실제 첫 검색이 대개 공급자 분류가 아니라 상황에서 출발하기 때문이다. 즉, 사용자는 보통 `키즈카페 유형별 목록`보다 `비 오는데 오늘 아이랑 어디 가지?`에 가깝게 검색한다.

## 콘텐츠 구조

### 1주차

1. `수도권 비 오는 날 아이와 갈 만한 실내 장소`
   - 역할: 클러스터 허브
   - 우선 키워드: `비오는날 아이와 가볼만한곳`, `비오는날 아이랑`, `아이와 실내 가볼만한곳`
   - 페이지 과업: 사용자가 어느 지역과 조건으로 더 좁힐지 결정하게 한다.

2. `서울 비 오는 날 아이랑 갈 만한 실내 장소`
   - 역할: 서울 지역 지원 글
   - 우선 키워드: `서울 비오는날 아이와`, `서울 아이랑 실내`, `서울 실내 아이와 가볼만한곳`
   - 페이지 과업: 강서, 송파, 광진, 도봉 등 생활권 기준으로 고르게 한다.

### 2주차

3. `비 오는 날 무료·저렴한 실내 나들이`
   - 역할: 비용 기준 지원 글
   - 우선 키워드: `무료 실내 아이와`, `아이와 무료 가볼만한곳`, `저렴한 키즈카페`
   - 페이지 과업: 무료, 부분 무료, 저비용 공공 시설을 먼저 고르게 한다.

4. `예약 없이 갈 수 있는 실내 아이 나들이`
   - 역할: 당일 선택 지원 글
   - 우선 키워드: `예약없이 아이와 갈만한곳`, `당일 아이랑 갈만한곳`, `주말 실내 아이랑`
   - 페이지 과업: 계획 없이 당일 갈 수 있는 후보를 빠르게 좁히게 한다.

### 3주차

5. `인천 비 오는 날 아이랑 갈 만한 실내 장소`
   - 역할: 인천 지역 지원 글
   - 우선 키워드: `인천 비오는날 아이와`, `인천 아이랑 실내`, `인천 실내 키즈카페`
   - 페이지 과업: 연수, 서구, 계양, 강화 등 권역별로 선택하게 한다.

6. `경기 남부 비 오는 날 아이랑 갈 만한 실내 장소`
   - 역할: 경기 남부 지역 지원 글
   - 우선 키워드: `경기 남부 아이와 실내`, `수원 아이랑 실내`, `성남 아이와 실내`, `용인 아이와 실내`
   - 페이지 과업: 검색 수요가 큰 경기 남부 생활권을 중심으로 선택하게 한다.

### 4주차

7. `1~3세 아이와 비 오는 날 갈 만한 실내 장소`
   - 역할: 영유아 연령 지원 글
   - 우선 키워드: `아기랑 실내 갈만한곳`, `돌아기 실내`, `3세 아이 실내`
   - 페이지 과업: 짧은 체류, 유모차, 낮은 자극, 빠른 이탈 가능성을 기준으로 고르게 한다.

8. `3~6세 아이와 비 오는 날 갈 만한 실내 장소`
   - 역할: 유아 연령 지원 글
   - 우선 키워드: `5세 아이 실내`, `유아 실내 체험`, `6세 아이와 갈만한곳`
   - 페이지 과업: 체험성, 체력 소모, 전시, 반복 놀이 가능성을 기준으로 고르게 한다.

경기 북부는 이번 첫 4주 클러스터에서는 허브 글 안의 대표 섹션으로 다룬다. 초기 데이터에서 `고양`, `파주`, `의정부`, `경기 북부 비오는날` 수요가 확인되면 후속 클러스터에서 별도 글로 분리한다.

## 글 템플릿

모든 글은 같은 편집 뼈대를 사용하되, 판단 기준만 다르게 둔다.

1. H1
   - `비 오는 날`, `비오는날`, `아이와`, `아이랑`, `실내`, 지역, 연령, 비용, 예약 중 2~3개를 자연스럽게 포함한다.

2. 첫 문단
   - 사용자의 상황을 바로 짚는다.
   - 실내 여부만으로는 부족하고 연령, 예약, 주차, 비용, 체류 시간을 함께 봐야 한다고 설명한다.

3. 한눈에 보는 요약 표
   - 컬럼: 장소, 지역, 권장 연령, 비용, 예약, 주차, 이럴 때 추천.

4. 추천 장소 카드
   - 장소 데이터가 충분하면 글당 8~12곳을 추천한다.
   - 사실 정보는 `content/places`를 기준으로 한다.
   - 카드 설명은 seed 문장 복사가 아니라 글의 판단 기준에 맞춰 다시 쓴다.

5. 고르는 기준
   - 허브 글: 지역과 이동 부담 기준.
   - 지역 글: 생활권과 이동 동선 기준.
   - 비용 글: 무료, 부분 무료, 저비용 기준.
   - 예약 글: 당일 가능성 기준.
   - 연령 글: 체류 난이도와 아이 적합도 기준.

6. 방문 전 확인 체크리스트
   - 운영시간
   - 휴무일
   - 예약 회차
   - 아이/보호자 요금
   - 주차
   - 확인 가능한 경우 수유실 또는 기저귀교환대

7. 관련 링크
   - `/places`
   - 관련 `/places/<region>`
   - 관련 조건 또는 연령 필터 URL
   - 비용 또는 공공시설 맥락이 있으면 관련 혜택 글
   - 장소와 매칭되는 기존 주차 글

## 중복 방지 규칙

- 모든 글은 하나의 대표 질문을 가진다.
- 대표 질문은 H1과 첫 문단에 보이게 한다.
- 같은 장소 설명 문장을 여러 글에 재사용하지 않는다.
- 허브 글은 짧은 비교 요약을 담당한다.
- 하위 글은 실제 선택 기준을 담당한다.
- 같은 장소가 여러 글에 등장하면 설명 포인트를 바꾼다.
  - 지역 글: 이동 동선과 생활권 적합성
  - 비용 글: 무료 또는 부분 무료 가치
  - 예약 글: 당일 가능성
  - 연령 글: 체류 난이도와 아이 적합도
- 운영 사실은 `content/places`에서 가져오고, 글은 판단과 맥락을 더한다.

## SEO 규칙

- 주 색인 표면은 글 URL로 둔다.
- `/places?rain=true`, `/places?indoor=true` 같은 필터 URL은 유용한 CTA로만 사용한다.
- `비 오는 날`과 `비오는날`을 제목, 도입부, 소제목, 본문에 자연스럽게 분산한다.
- 메타 설명은 80~120자 안에서 지역, 상황, 선택 기준을 담는다.
- 키워드를 억지로 나열한 제목과 소제목을 피한다.
- 모든 글에 `기준일: 2026년 6월`을 넣는다.
- 변동 가능한 사실에는 공식 출처 확인 안내를 붙인다.
- 사용자가 같은 질문으로 바로 다시 검색하지 않아도 될 만큼 충분히 답한다.

## 내부 링크 규칙

- 허브 글은 7개 하위 글을 모두 링크한다.
- 모든 하위 글은 허브 글로 되돌아간다.
- 모든 하위 글은 관련 지역 페이지 1개 이상을 링크한다.
- 모든 하위 글은 관련 조건 또는 연령 글 1~2개를 링크한다.
- 비용 글은 관련 혜택 글로 연결한다.
- 예약 글은 유용한 `/places` 필터로 연결한다.
- 연령 글은 해당 연령 `/places` 필터로 연결한다.
- 기존 주차 글은 매칭되는 장소의 보조 링크로 사용한다.

## 데이터와 검증 정책

장소 사실 정보의 소스 오브 트루스는 `content/places`다.

대표 추천에 사용할 수 있는 상태:

- `official_verified`
- `semi_verified`

대표 추천에서 제외할 상태:

- `discovery_only`
- `needs_refresh`

발행 규칙:

- 운영시간, 요금, 예약 상태, 주차 같은 변동 정보는 영구적인 사실처럼 단정하지 않는다.
- 바뀔 수 있는 세부 정보에는 `방문 전 공식 페이지 확인` 같은 표현을 사용한다.
- 공식 근거가 약한 상업형 키즈카페는 최상단 대표 장소가 아니라 대안 후보로 낮춰 배치한다.
- 모든 글에는 `기준일: 2026년 6월`을 넣는다.

## 측정 기준

4주 후 우선 확인할 지표:

- 8개 글 URL별 Search Console 노출 수
- 넓은 키워드, 지역 키워드, 비용 키워드, 예약 키워드, 연령 키워드의 쿼리 다양성
- 글에서 `/places`로 이동한 클릭
- 글에서 지역별 장소 페이지로 이동한 클릭
- 글에서 혜택 또는 주차 보조 글로 이동한 클릭

성공 신호:

- 허브 글이 넓은 비 오는 날 실내 키워드 노출을 받는다.
- 하위 글이 서로 다른 지역, 비용, 예약, 연령 쿼리를 받는다.
- 사용자가 글에서 `/places` 또는 관련 페이지로 이동한다.

실패 신호:

- 8개 글이 모두 같은 소수 키워드만 두고 경쟁한다.
- 하위 글이 구분된 노출을 받지 못한다.
- 글에서 장소 페이지로 이동하는 행동이 약하다.
- 사용자가 글에 들어온 뒤 유용한 다음 행동 없이 이탈한다.

## 구현 메모

새 육아 가이드 카테고리를 도입하면 아래를 갱신한다.

- `lib/blog/posts.ts`
- 카테고리 표시와 목록 동작
- 사이트맵 노출
- 필요한 경우 `/places`와 홈 내부 링크

기존 카테고리 아래에 글을 넣는다면 장기 리브랜딩 taxonomy와 충돌하지 않는 이름을 사용한다.

글만 추가하는 경우 우선 확인할 것:

- 마크다운 렌더링
- sitemap, canonical, RSS 반영
- 코드나 카테고리 매핑이 바뀌면 `pnpm type-check`

공통 글 카드나 장소 렌더링 코드를 수정하는 경우에는 구현 세부가 아니라 동작을 검증하는 테스트를 추가한다.

## 승인 기록

사용자가 승인한 항목:

1. 상황형 피라미드 콘텐츠 구조
2. 글별 역할 분리와 중복 방지 규칙
3. 4주 발행 순서와 우선 키워드
4. 글 템플릿, SEO 규칙, 내부 링크 규칙

## 간단한 결론

가장 빠르고 안전한 경로는 개별 글을 흩뿌리는 것이 아니라, 비 오는 날 실내 콘텐츠 클러스터를 집중 발행하는 것이다. 기존 장소 seed를 기반으로 허브 글 1개와 하위 글 7개를 만들고, 지역·비용·예약·연령 검색 의도를 나눠 받는 구조로 운영한다.

# Roadmap And Measurement

## 목표

리브랜딩을 한 번에 끝내려 하지 않고, 수도권 중심 단계적 론칭과 확장 순서를 문서화한다.

## Phase A, 수도권 론칭

범위:

- 서울
- 경기 북부
- 경기 남부
- 인천

우선 산출물:

- 홈 리브랜딩
- `아이와 가볼 곳` 허브
- 핵심 도구 2~3개
- 혜택 허브 기본 구조
- 육아형 카테고리 재정의

### Phase A 공급 전략

Phase A는 지역을 넓게 여는 것보다 `공급 난이도별로 다르게 시작`한다.

1. `서울 공공형 심층 허브`
   - 공공서비스예약, 육아종합지원센터, 박물관/체험관 등 공식 소스 중심
2. `경기 남부 상업형 제한 허브`
   - 수동 검증 가능한 키즈카페/체험시설 10~15개 수준으로 시작
3. `경기 북부/인천`
   - 허브 구조와 템플릿은 준비하되, 깊이는 위 두 버킷 이후 확대

이렇게 해야 수동 검증 모델과 초기 운영 역량이 맞는다.

## Phase B, 광역시 확장

범위:

- 부산
- 대구
- 대전
- 광주
- 세종

추가 산출물:

- 지역 확장용 템플릿
- 콘텐츠 운영 규칙 자동화
- 도구 확장

## Phase C, 전국 확장

범위:

- 전국 지역 구조
- 조건별 랜딩 강화
- 장소 카드 데이터 구조 고도화

## 핵심 지표

### North Star

`주간 계획 세션 수`

정의:

- 사용자가 장소 글 또는 장소 허브에 진입하고
- 같은 세션 안에서 도구 클릭, 관련 글 이동, 혜택 진입 중 하나 이상을 수행한 세션 수

이 지표가 중요한 이유는 단순 페이지뷰가 아니라 `실제 계획 행동`을 잡기 때문이다.

### 측정 축

1. 검색 유입 증가
2. Places 진입 후 2차 행동률
3. 7일 재방문 사용자 수
4. 주간 재방문율
5. 도구 재사용 세션 비율
6. 지역 허브 유입 비중
7. 도구 사용 후 관련 글 이동률

### Owned Loop 지표

Phase A에서 최소 하나의 owned loop를 둔다.

후보:

- 카카오 채널 구독
- 이메일 뉴스레터 구독
- 주간 큐레이션 링크 재방문

최소 지표:

- owned capture conversion rate
- owned channel 재방문 세션 비중

## 분석 연결 포인트

현재 저장소 기준으로 추후 점검 위치:

- GA 로직: `lib/analytics/ga4.ts`
- 방문자 API: `app/api/analytics/visitors/route.ts`

## 운영 cadence 제안

검증 비용을 감안해 Phase A cadence를 공급 유형별로 분리한다.

- 주 1개 서울 공공형 글
- 주 1개 경기 남부 상업형 글
- 격주 1개 혜택/절약형 글
- 월 1개 핵심 wedge 도구 추가 또는 개선

무리해서 글 수를 늘리기보다 stale data를 줄이는 쪽이 우선이다.

## 주요 리스크

1. 초기 범위가 넓어짐
2. 카테고리 이름이 계속 바뀜
3. 도구가 콘텐츠와 분리됨
4. 혜택 글 신뢰도가 떨어짐
5. 재방문 루프 없이 SEO 허브로만 남을 수 있음
6. 상업형 시설 수동 검증이 발행 cadence를 잡아먹을 수 있음

## 리스크 대응

1. 수도권 외 구조는 문서상으로만 정의하고 즉시 공개하지 않음
2. 카테고리명과 메타 필드를 먼저 고정
3. 모든 도구는 관련 글 세트를 먼저 가진 상태에서 공개
4. 혜택 글은 기준 날짜와 출처 점검을 발행 조건으로 둠
5. 카카오 채널 또는 이메일 중 최소 하나를 Phase A owned loop로 둠
6. 서울 공공형과 경기 남부 상업형을 분리 운영해 검증 부담을 조절함

간단한 결론:

이 리브랜딩은 전국 포털을 한 번에 여는 작업이 아니라, 수도권에서 깊이를 만든 뒤 넓히는 작업으로 운영해야 한다.

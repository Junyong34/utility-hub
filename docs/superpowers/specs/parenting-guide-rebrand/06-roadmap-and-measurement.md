# Roadmap And Measurement

## 목표

리브랜딩을 한 번에 끝내려 하지 않고, 수도권 중심 단계적 론칭과 확장 순서를 문서화한다.

## Phase A, 수도권 론칭

범위:

- 서울
- 경기 북부
- 경기 남부
- 인천

우선 산출물:

- 홈 리브랜딩
- `아이와 가볼 곳` 허브
- 핵심 도구 2~3개
- 혜택 허브 기본 구조
- 육아형 카테고리 재정의

## Phase B, 광역시 확장

범위:

- 부산
- 대구
- 대전
- 광주
- 세종

추가 산출물:

- 지역 확장용 템플릿
- 콘텐츠 운영 규칙 자동화
- 도구 확장

## Phase C, 전국 확장

범위:

- 전국 지역 구조
- 조건별 랜딩 강화
- 장소 카드 데이터 구조 고도화

## 측정 축

1. 검색 유입 증가
2. 글 → 도구 클릭률
3. 세션당 페이지 수
4. 지역 허브 유입 비중
5. 도구 사용 후 관련 글 이동률

## 분석 연결 포인트

현재 저장소 기준으로 추후 점검 위치:

- GA 로직: `lib/analytics/ga4.ts`
- 방문자 API: `app/api/analytics/visitors/route.ts`

## 운영 cadence 제안

- 주 2~3개 지역/테마 글
- 월 1~2개 핵심 도구 추가 또는 개선
- 월 2개 이상 혜택/절약형 글 갱신

## 주요 리스크

1. 초기 범위가 넓어짐
2. 카테고리 이름이 계속 바뀜
3. 도구가 콘텐츠와 분리됨
4. 혜택 글 신뢰도가 떨어짐

## 리스크 대응

1. 수도권 외 구조는 문서상으로만 정의하고 즉시 공개하지 않음
2. 카테고리명과 메타 필드를 먼저 고정
3. 모든 도구는 관련 글 세트를 먼저 가진 상태에서 공개
4. 혜택 글은 기준 날짜와 출처 점검을 발행 조건으로 둠

간단한 결론:

이 리브랜딩은 전국 포털을 한 번에 여는 작업이 아니라, 수도권에서 깊이를 만든 뒤 넓히는 작업으로 운영해야 한다.

# Content Model And Taxonomy

## 목표

블로그와 장소 정보가 산만해지지 않도록 콘텐츠 유형, 분류 축, 메타 필드를 고정한다.

## 콘텐츠 기본 단위

`SEO 글 중심 + 표준 장소 카드 혼합형`

원칙:

1. 검색 유입을 받는 단위는 글이다.
2. 글 안에는 반복 가능한 장소 카드 블록을 넣는다.
3. 카드 구조는 나중에 장소 DB로 확장 가능해야 한다.

## 1차 카테고리

1. 지역별 가이드
2. 테마별 가이드
3. 문제 해결형 가이드

## 추천 제목 패턴

- `수원 24~48개월 실내 체험 7곳`
- `서울 무료 키즈 시설 모음`
- `비 오는 날 성남에서 갈 만한 곳`
- `용인 5세 아이와 주말 당일치기 코스`

## 메타 필드

글과 카드 공통 메타:

- 지역
- 연령대
- 실내/야외
- 예산대
- 날씨 적합도
- 테마
- 동반 조건

`동반 조건` 예시:

- 조부모 동반 가능
- 유모차 이동 편함
- 주차 편함

## 표준 장소 카드 필드

필수:

- 장소명
- 지역
- 권장 연령
- 실내/야외
- 가격대
- 핵심 포인트 한 줄

권장:

- 주차
- 수유실
- 유모차
- 우천 대체 가능 여부
- 체류 권장 시간

## 에디토리얼 규칙

글 발행 시 체크리스트:

1. 관련 지역 글 링크 1개 이상
2. 관련 도구 링크 1개 이상
3. 관련 혜택 글 링크 1개 이상
4. 표준 카드 블록 사용
5. 메타 필드 입력

## 제외할 콘텐츠

- 후기 모음형 글
- 개인 육아 에세이
- 전국 단위 얕은 리스트
- 정보 출처가 불명확한 지원금 재가공 글

## 현재 저장소 반영 포인트

- 카테고리 매핑: `lib/blog/posts.ts`
- 블로그 UI: `components/blog/*`
- 포스트 원본 위치: `content/posts/<category>/*.md`

## 후속 작업

- 육아용 신규 카테고리 슬러그 설계
- frontmatter 필드 확장
- 표준 장소 카드 마크다운 패턴 정의

간단한 결론:

콘텐츠는 글이 유입을 잡고, 카드가 구조화를 담당하는 방식으로 운영해야 한다.

# Calculator Design And Visualization

## 목표

아이 관련 계산기를 어떤 공식으로 만들고, 어떤 계산기에 차트를 붙일지 기준을 고정한다.

## 계산기 설계 원칙

1. 공식이나 법령 근거가 있는 계산기는 근거 문서를 명시한다.
2. 제도 변경 가능성이 있는 계산기는 정책 버전 파일로 분리한다.
3. 차트는 `멋`이 아니라 `해석`에 도움이 될 때만 넣는다.
4. 결과가 숫자 1개면 차트를 기본값으로 붙이지 않는다.

## 1차 계산기 후보와 설계 기준

### 1. 연령/개월수 계산기

#### 목적

- 만 나이
- 개월수
- 연령대 태그
- 어린이집/유치원/학교 관련 콘텐츠 연결

#### 근거

- 국가법령정보센터의 민법 제158조  
  나이는 출생일을 산입해 만 나이로 계산하고, 1세 미만은 월수로 표시 가능

#### 계산 방식

- 만 나이: `기준일 연도 - 출생 연도`, 생일 미도래 시 `-1`
- 개월수: `연도 차이 * 12 + 월 차이`, 기준일의 일이 출생일보다 작으면 `-1`

#### 차트

- 기본은 차트 없음
- 대신 `타임라인 배지` 또는 `연령 구간 바` 정도만 허용

### 2. 성장 계산기

#### 목적

- 키/몸무게/성별/생년월일 기준 성장 상태를 참고용으로 보여줌
- 부모가 변화 추이를 기록할 수 있게 함

#### 근거

- 질병관리청 소아청소년 성장도표를 기준으로 설계

#### 계산 방식

- 1차는 `백분위` 기준 표시
- 가능하면 연령/성별별 공식 표를 JSON 정책 데이터로 분리
- `진단` 표현은 금지하고 `참고용 성장 위치`로 표현

#### 차트

- 필요
- 권장 차트:
  - 현재 값 점 + 백분위 기준선이 있는 선형 차트
  - 여러 측정값 입력 시 추이 선 그래프

### 3. 자녀 비과세 증여 계산기

#### 목적

- 자녀에게 얼마를 증여할 때 과세표준과 예상 세액이 어떻게 되는지 간단히 보기

#### 근거

- 상속세 및 증여세법
- 홈택스 모의계산

#### 계산 방식

- 증여재산공제
- 10년 합산 기준
- 과세표준 구간별 세율/누진공제

정확한 수치는 정책 버전 파일에서 관리한다.

#### 차트

- 필요
- 권장 차트:
  - `증여액 → 공제 → 과세표준 → 예상세액` 워터폴형 막대

### 4. 가족 나들이 예산 계산기

#### 목적

- 교통, 입장료, 식비, 주차비, 간식비 등으로 하루 예산 가늠

#### 근거

- 공식 법령이 아니라 사용자 입력 기반 계산

#### 계산 방식

- 항목별 입력값 합산
- 인원수 기준 보정
- 고정비/변동비 분리 가능

#### 차트

- 필요
- 권장 차트:
  - 카테고리별 비중 도넛
  - 항목별 누적 스택 바

### 5. 키즈카페/체험 월예산 계산기

#### 목적

- 월별 외출 빈도와 평균 단가를 넣어 월 소비 수준 파악

#### 근거

- 공식 법령 없음, 사용자 입력 기반

#### 계산 방식

- `월 방문 횟수 * 회당 평균 비용`
- 정기권/1회권 비교 옵션

#### 차트

- 필요
- 권장 차트:
  - `1회권 vs 정기권` 비교 막대
  - 월간 비용 분해 스택 바

### 6. 지역 혜택 자가진단 도구

#### 목적

- 사용자의 지역, 자녀 연령, 가구 상황을 기준으로 어떤 혜택 페이지를 먼저 봐야 하는지 빠르게 안내

#### 근거

- 공식 법령 계산기가 아니라 규칙 기반 분기 도구
- 실제 금액 계산보다 `해당 가능성이 높은 혜택 묶음`을 추천하는 역할

#### 계산 방식

- 지역
- 자녀 연령대
- 가구 상황
- 보호자 상태

위 입력을 바탕으로 관련 혜택 글과 체크리스트를 반환한다.

#### 차트

- 기본은 차트 없음
- 대신 우선순위 카드, 체크리스트, 태그형 결과를 사용

### 7. 육아휴직 급여 계산기

#### 목적

- 육아휴직 급여 예상액과 월별 상한 반영 결과 표시

#### 근거

- 고용노동부 공식 안내
- 제도 변경 시점 기준 버전 관리 필요

#### 계산 방식

- 통상임금 기준
- 일반 육아휴직 급여
- `6+6 부모육아휴직제` 분기 로직
- 상한/하한을 정책 테이블로 관리

#### 차트

- 선택
- 권장 차트:
  - 월별 지급 예상액 막대

## 차트 사용 기준

### 차트를 써야 하는 경우

- 추세를 보여줘야 할 때
- 구성 비율을 보여줘야 할 때
- 정책 적용 전후 비교가 필요할 때
- 누적 구조를 설명해야 할 때

### 차트를 쓰지 말아야 하는 경우

- 결과가 숫자 1개일 때
- 사용자가 바로 이해 가능한 단순 합계일 때
- 모바일에서 오히려 읽기 어려울 때

## 시각화 라이브러리 기준

현재 저장소에는 `recharts` 의존성이 이미 있다.

1차 원칙:

- 차트는 `Recharts`
- 숫자 해석이 먼저, 애니메이션은 최소
- 모바일에서 가로 스크롤 없는 구조 우선

## 정책 버전 파일 원칙

법/제도 기반 계산기는 아래 구조로 설계한다.

- `lib/tools/<tool>/policies.ts`
- 시행일
- 정책 이름
- 상한/하한
- 세율/공제/구간표
- 버전 주석

컴포넌트에 수치를 하드코딩하지 않는다.

## 추천 시작 순서

첫 계산기는 아래 순서가 맞다.

1. 연령/개월수 계산기
2. 가족 나들이 예산 계산기
3. 키즈카페·체험 월예산 계산기
4. 지역 혜택 자가진단 도구
5. 성장 계산기
6. 자녀 비과세 증여 계산기
7. 육아휴직 급여 계산기

이 순서의 이유:

- 빠르게 출시 가능
- 검증이 쉬움
- 장소/혜택 콘텐츠와 직접 연결됨
- 정책 복잡도가 단계적으로 올라감

## 신뢰도 메모

법/제도 계산기는 반드시 공식 소스와 모의계산 결과를 함께 대조해야 한다.  
특히 증여세, 육아휴직 급여는 제도 변경 가능성이 있어 현재 수치가 바뀔 수 있다. 이 부분은 정확하지 않을 수 있으므로 구현 직전 재검증이 필요하다.

간단한 결론:

첫 계산기는 `연령/개월수`와 `나들이 예산`부터 시작하고, 정책형 계산기는 버전 파일 기반으로 설계해야 한다.

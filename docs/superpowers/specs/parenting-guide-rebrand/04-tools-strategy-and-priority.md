# Tools Strategy And Priority

## 목표

도구를 보조 기능이 아니라 메인 전환 축으로 운영하되, 장소 경험을 방해하지 않는 구조를 정한다.

## 도구 포지셔닝

도구는 두 번째 메인 축이다.

사용자 흐름:

1. 장소 글에서 문제 인식
2. 관련 도구에서 계산/판단
3. 다시 장소 또는 혜택 콘텐츠로 이동

## Phase A 핵심 wedge 도구

Phase A 상위 도구는 `이번 주말 아이와 어디 가지?`라는 핵심 JTBD를 직접 강화해야 한다.

1. 연령/개월수 계산기
2. 가족 나들이 예산 계산기
3. 키즈카페·체험 월예산 계산기
4. 지역 혜택 자가진단 도구

이 네 개는 장소 탐색 직후 바로 쓰이거나, 장소 선택을 더 쉽게 만드는 도구들이다.

## Phase A 보조 도구

아래 도구는 유용하지만, 핵심 wedge보다는 한 단계 뒤에 둔다.

1. 성장 계산기
2. 자녀 비과세 증여 계산기
3. 어린이집·유치원 비용 계산기

## Phase B 후보

- 육아휴직 급여 계산기
- 주말 나들이 비용 체크리스트
- 지역 혜택 자가진단 고도화

## 도구 페이지 공통 규칙

1. 독립 SEO 랜딩으로 읽혀야 한다.
2. 관련 글 2~3개 이상과 연결한다.
3. 결과 화면에 다음 행동 링크를 둔다.
4. 입력값은 모바일에서도 빠르게 조작 가능해야 한다.
5. Phase A 상위 도구는 장소 허브에서 바로 진입 가능한 문맥을 가져야 한다.

## 제외 규칙

초기에는 아래를 하지 않는다.

- 의료 진단처럼 오해될 수 있는 계산기
- 법률 자문처럼 보이는 결과 표현
- 데이터 유지보수 비용이 과도한 외부 API 의존 도구

## 현재 저장소 반영 포인트

- 도구 레지스트리: `lib/tools/tool-config.ts`
- 라우트 패턴: `app/tools/<tool-id>/page.tsx`
- UI 구현: `components/tools/**`

## 후속 작업

- Phase A 핵심 wedge 도구 4개 상세 PRD 작성
- Phase A 보조 도구 3개 상세 PRD 작성
- 각 도구와 연결될 글 세트 정의
- 도구 허브 카테고리와 카드 스타일 정의

간단한 결론:

도구는 별도 섬이 아니라 장소와 혜택 콘텐츠를 더 깊게 소비하게 만드는 연결 장치로 설계해야 한다.

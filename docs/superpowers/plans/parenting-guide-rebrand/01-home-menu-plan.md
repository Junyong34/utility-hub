# Home Menu Plan

## 목표

홈을 `검색 + 큐레이션 혼합형` 구조로 만들고, 사용자가 `장소`, `도구`, `혜택` 중 원하는 경로를 바로 선택하게 한다.

## 역할

홈은 브랜드 소개 페이지가 아니다.
홈은 `첫 선택을 빠르게 도와주는 진입 페이지`다.

## 우선순위

1. 장소 탐색 시작
2. 도구 진입
3. 혜택 진입
4. 최신 글 재탐색

## 섹션 구조

1. 히어로
2. 이번 주 인기 장소
3. 지역별 바로가기
4. 상황별 바로가기
5. 도구 모음
6. 혜택·지원금 모음
7. 최신 가이드

## 히어로 요구사항

- H1과 서브카피는 실용형 톤 유지
- 검색 입력 포함
- 지역/연령/상황 빠른 필터 포함
- 모바일에서도 한 화면 안에서 첫 행동이 가능해야 함

## 필요한 데이터/콘텐츠

- 지역별 대표 카드
- 상황별 대표 카드
- 도구 대표 카드
- 혜택 대표 카드
- 최신 글 리스트

## 연결 규칙

- 장소 카드 클릭 시 관련 글 또는 허브로 이동
- 도구 카드 클릭 시 `/tools`
- 혜택 카드 클릭 시 `/benefits`

## 구현 전 결정할 것

- 홈 검색이 허브 검색인지, 블로그 검색인지
- 인기 장소 선정 기준
- 홈에 노출할 카드 개수

간단한 결론:

홈은 브랜드 설명보다 `빠른 선택`이 먼저여야 한다.

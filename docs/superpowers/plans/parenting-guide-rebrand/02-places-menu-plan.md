# Places Menu Plan

## 목표

`아이와 가볼 곳` 메뉴를 리브랜딩의 핵심 허브로 만든다.

## 역할

사용자가 지역과 조건을 기준으로 갈 곳을 찾는 메인 허브다.

## 1차 구조

- `/places`
- `/places/seoul`
- `/places/gyeonggi-north`
- `/places/gyeonggi-south`
- `/places/incheon`

## 2차 탐색 축

- 연령
- 실내/야외
- 무료/유료
- 날씨/상황

## 노출 방식

1. 지역 허브
2. 조건별 큐레이션
3. 대표 글 묶음
4. 표준 장소 카드

## 콘텐츠 요구사항

- 지역별 대표 글 세트
- 상황별 대표 글 세트
- 장소 카드 블록 표준화

## 연결 규칙

- 장소 글에서 관련 도구 링크 포함
- 장소 글에서 관련 혜택 링크 포함
- 지역 허브에서 대표 도구 1개 이상 노출

## 구현 전 결정할 것

- 지역 슬러그 한글/영문 정책
- `/places`와 기존 `/blog` 연결 방식
- 표준 장소 카드 렌더링 방식

간단한 결론:

`아이와 가볼 곳`은 이 리브랜딩의 중심 메뉴고, 가장 먼저 설계와 구현이 시작돼야 한다.

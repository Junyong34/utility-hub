# Benefits Menu Plan

## 목표

혜택·지원금 메뉴를 `공지형 정보 모음`이 아니라 `실제 절약 가이드 허브`로 만든다.

## 역할

부모가 놓치기 쉬운 지원금, 지역 혜택, 절약 방법을 같은 맥락에서 찾게 한다.

## 1차 구성

- 정부지원
- 지역 혜택
- 절약 가이드

## 허브 요구사항

- 기준 날짜 노출
- 대표 글 카드
- 관련 도구 연결
- 관련 장소 글 연결

## 콘텐츠 원칙

- 누가 받을 수 있는지 명확히 설명
- 얼마가 절약되는지 체감 가능하게 표현
- 정책 나열보다 사용 맥락 중심

## 구현 전 결정할 것

- `/benefits` 하위 카테고리 구조
- 기준 날짜/출처 UI
- 지역 혜택 데이터 관리 방식

간단한 결론:

혜택 메뉴는 트래픽용 정보창고가 아니라, 장소와 도구 경험을 보완하는 실용 허브여야 한다.

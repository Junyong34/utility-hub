# Blog Menu Plan

## 목표

블로그 메뉴를 단순 아카이브가 아니라, 전체 콘텐츠 지형을 보여주는 정리 허브로 재구성한다.

## 역할

카테고리와 최신 글을 정리하고, 사용자가 검색 외 방식으로도 콘텐츠를 다시 탐색하게 한다.

## 1차 구조

- 지역별 가이드
- 테마별 가이드
- 문제 해결형 가이드

## 블로그 허브 요구사항

- 카테고리별 진입
- 최신 글
- 인기 글
- 관련 도구 연결
- 관련 혜택 연결

## 구현 전 결정할 것

- 기존 카테고리 정리 방식
- 신규 육아 카테고리 슬러그
- 블로그 허브와 `아이와 가볼 곳` 허브의 역할 분리

## 원칙

- 블로그는 운영 아카이브
- `아이와 가볼 곳`은 사용자 과업 허브
- 둘의 역할은 겹치되, 목적은 다르게 둔다

간단한 결론:

블로그 메뉴는 글 모음을 보여주고, 핵심 탐색은 `아이와 가볼 곳` 허브에서 담당하게 해야 한다.

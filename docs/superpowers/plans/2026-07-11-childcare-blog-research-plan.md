1. `어린이집 입소대기 신청은 어디서 하나?`
2. `신청 전 먼저 확인할 기본 조건`
3. `아이사랑 입소대기 신청 흐름`
4. `어린이집 정보공개포털에서 봐야 할 항목`
5. `정원·현원·대기순번은 어떻게 해석할까?`
6. `평가, 보육과정, 급식·안전 정보 확인법`
7. `연장보육·통학차량·운영시간 체크포인트`
8. `현장 상담 때 꼭 물어볼 질문`
9. `지역별 보육포털과 어린이집별 차이 주의사항`
10. `FAQ`

체크리스트 표 초안:

| 확인 항목 | 공식 확인 위치 | 왜 중요한가 | 현장 상담 질문 |
|---|---|---|---|
| 정원·현원·대기자수 | 아이사랑 어린이집찾기 | 실제 입소 가능성을 가늠 | 올해 중도 입소 가능성이 있나요? |
| 운영시간·연장보육 | 정보공개포털, 연장보육반 검색 | 복직·퇴근 시간과 직결 | 연장반은 매일 운영되나요? |
| 교직원·반 현황 | 정보공개포털 공시항목 | 반 구성과 보육 안정성 확인 | 담임 교사 변경이 잦은 편인가요? |
| 급식·안전·통학버스 | 정보공개포털 공시항목 | 안전과 등하원 동선 확인 | 통학차량 시간과 안전교육 기록은 어떻게 확인하나요? |

FAQ 후보: 어린이집 입소대기는 선착순인가요, 아이사랑에서 몇 곳까지 신청할 수 있나요, 대기순번이 앞인데도 입소가 안 될 수 있나요, 연장보육반은 모든 어린이집에서 운영하나요, 입소 확정 후 다른 대기 신청은 자동 취소되나요.

## Writing Tasks

- [ ] 각 주제 발행 직전 공식 페이지를 다시 열어 기준일을 최신화한다.
- [ ] 각 글의 frontmatter에 `date`, `author`, `excerpt`, `tags`, `category`, `categorySlug`를 채운다.
- [ ] 글 상단에 기준일과 정책 변경 가능성 문구를 넣는다.
- [ ] 본문에 최소 1개 이상의 비교표 또는 체크리스트 표를 넣는다.
- [ ] FAQ를 5개 이상 넣고, 질문에는 실제 검색 쿼리형 문장을 사용한다.
- [ ] 민간 블로그 벤치마킹에서 얻은 경험형 질문은 FAQ와 체크리스트에 녹이되, 사실 근거로 인용하지 않는다.
- [ ] 관련 기존 글과 `/benefits` 허브로 내부 링크를 연결한다.
- [ ] 발행 후 `pnpm format:check`를 실행한다.

## Do Not Use

- `맞벌이는 무조건 지원`
- `신청하면 바로 매칭`
- `당일 돌봄은 반드시 가능`
- `1시간만 이용 가능`
- `어린이집 대기순번이 앞이면 반드시 입소`
- `모든 어린이집이 연장보육 운영`

간단한 결론:

세 글 모두 단순 정보 나열보다 `돈`, `당일 가능성`, `대기 신청 전 체크리스트`처럼 부모가 바로 판단해야 하는 질문을 앞에 두는 편이 검색 유입과 체류에 유리하다.

# 04차: 페이지별 라우터 정의 및 SSG/SSR 설계

> 상세화 전 검토 체크포인트  
> - SSG/SSR 분류 기준이 콘텐츠 특성과 운영 목적에 맞는가?  
> - 라우터별 데이터 소스와 캐싱 전략의 충돌이 없는가?  
> - 광고 정책/SEO 우선순위가 라우트 단위로 반영 가능한가?

## 목적

- 페이지 유형별 라우팅 구조를 정의한다.
- SSG 성향 페이지와 SSR 성향 페이지를 구분해 렌더링 전략을 고정한다.
- 이후 상세 페이지 설계에서 재사용할 라우트 매트릭스를 제공한다.

## 범위(포함/제외)

- 포함:
  - 라우트 목록과 렌더 모드 정의
  - 라우트별 데이터 소스/SEO 우선순위 정의
  - 캐싱/갱신 정책의 기본 원칙 정의
- 제외:
  - 실제 라우트 파일 구현
  - 페이지 내부 컴포넌트 상세 구현

## 선행조건

- 02차 IA 확정, 03차 광고 슬롯 후보 정의가 필요하다.

## 핵심 작업

- 핵심 라우트 식별 및 그룹화
- SSG/SSR 선택 기준을 문서화
- 라우트별 데이터 소스 특성 정리
- SEO/광고 정책 고려사항을 라우트 단위로 연결

## 산출물

- SSG/SSR 라우트 분류 매트릭스
- 캐싱/렌더링 정책 기준표
- 상세 설계 입력용 라우트 우선순위 목록

## 의사결정 포인트

- `/tools` 허브 페이지의 렌더 모드 선택
- 블로그 목록/상세의 정적 재생성 주기
- 계산기 결과의 URL 공유 정책

## 리스크

- 렌더 모드 오분류 시 성능/인덱싱 효율 저하
- 동적 데이터 의존 증가 시 SSR 비용 확대

## 완료 기준

- 핵심 라우트가 누락 없이 표로 정의됨
- SSG/SSR 기준과 예외 기준이 문서화됨
- 다음 단계 상세 설계에서 그대로 참조 가능한 상태임

## 다음 단계 인계

- 05차 페이지별 상세 설계에 필요한 라우트 정의 전달
- SEO/광고 관련 우선순위 태그 전달

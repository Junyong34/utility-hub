# 공개 제품 계약 매트릭스

기준일: 2026-07-13. 이 문서는 구조 이관 전 현재 코드에서 읽을 수 있는 공개 URL, query/default, status/shape/cache, SEO 노출 계약의 기준선이다. 새 모듈로 이관할 때 동작을 개선하지 않고 이 기준선을 먼저 보존한다.

- **명시 없음**: 해당 route 코드가 값을 직접 선언하지 않는다. 프레임워크 기본 동작을 계약으로 추정하지 않는다.
- **현재 코드 확인 필요**: 정적 코드만으로 실제 배포 응답을 확정할 수 없어 런타임 contract test가 필요하다.
- 모든 page는 `app/layout.tsx`의 `WebSite`, `Organization` JSON-LD를 공통으로 받는다. 표의 JSON-LD는 page 전용 추가 schema만 적는다.
- `generateMetadata`, `generateStaticParams`, `runtime`, `dynamic`, HTTP method export는 대응 `app` 파일에 남긴다.

### Page 계약 주의사항

- 등록된 공개 tool ID와 노출 순서의 source of truth는 각 tool의 `domain/manifest.ts`를 명시적으로 조합하는 `modules/tools/catalog/public.ts`다: `loan-calculator`, `dsr-calculator`, `savings-calculator`, `lotto`, `last-digit-game`, `pomodoro`, `home-buying-funds-calculator`.
- `/tools/home-check`와 `/tools/og-image-studio`는 tool registry에 없으므로 자동 sitemap 대상이 아니다.
- `/finance/**`는 noindex/nofollow만 확인된다. 현재 page/route 코드에서 인증·인가가 확인되지 않으므로 “private”를 접근 제어 보장으로 해석하지 않는다.
- canonical/OG가 page에서 미선언된 noindex page의 실제 parent metadata 병합 결과는 현재 코드 확인이 필요하다.

## 이관 검증 원칙

1. 정적 fixture는 URL, tool ID, category, query key/default, storage key, status/shape/header를 비교한다.
2. page/API contract test는 실제 `app` adapter를 통과해 관찰 가능한 결과를 검증한다.
3. metadata·sitemap·RSS는 생성 결과를 비교하고, 프레임워크가 추가하는 header는 production build/runtime smoke로 보완한다.
4. 표에서 `현재 코드 확인 필요`인 항목은 추정값으로 고정하지 않고 먼저 런타임 baseline을 캡처한다.

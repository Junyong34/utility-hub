# Data Sourcing And Trust Model

## 목표

키즈카페, 공공 놀이시설, 체험시설, 혜택 정보를 어디서 가져오고 어떤 순서로 검증할지 고정한다.

핵심 원칙:

1. 검색량보다 신뢰도를 먼저 본다.
2. 상업 시설과 공공 시설은 같은 방식으로 수집하지 않는다.
3. `데이터 수집 → 검증 → 정규화 → 발행 → 재검수` 순서를 고정한다.

## 소스 우선순위

### Level 1, 공식 1차 소스

가장 먼저 보는 소스다.

- 공식 웹사이트
- 공식 예약 페이지
- 공식 운영기관 페이지
- 정부/지자체 공식 페이지
- 공공데이터 포털의 공식 데이터셋

### Level 2, 공식/준공식 2차 소스

Level 1이 부족할 때 보완용으로 사용한다.

- 공공기관 운영 플랫폼
- 한국관광공사 등 공공기관 API
- 중앙/지방 육아지원기관 포털
- 공공서비스 예약 시스템

### Level 3, 민간 발견 소스

상업형 시설 탐색용이다. 사실 확인 없이 바로 발행하지 않는다.

- 네이버 플레이스
- 카카오맵
- 구글맵
- 운영사 SNS

## 시설 유형별 수집 전략

### 1. 상업형 키즈카페

현 시점 기준, 전국 상업형 키즈카페를 안정적으로 커버하는 최신 국가 단일 소스는 확인하지 못했다.  
검색 결과로 확인된 공공데이터포털의 `키즈카페 이용자 소비 현황 데이터`는 지역 한정이며 2022년 이후 업데이트 중단 안내가 있다. 이 판단은 현재 확인한 범위 기준이며, 완전히 정확하지 않을 수 있다.

따라서 운영 전략은 아래 순서가 맞다.

1. 민간 지도 서비스로 후보 수집
2. 공식 웹사이트/예약 페이지/SNS로 영업 여부와 가격 확인
3. 필요 시 전화 또는 최근 공지 확인
4. 내부 발행용 표준 카드로 정규화

권장 수집 항목:

- 시설명
- 주소
- 운영 시간
- 가격
- 연령 추천
- 주차
- 수유실/유모차/실내 여부
- 공식 확인 링크
- 마지막 검증일

### 2. 국가/지자체 제공 놀이시설

이 유형은 공공 소스를 1차로 본다.

우선 소스:

- 서울시 공공서비스예약
- 각 지자체 예약 시스템
- 육아종합지원센터
- 장난감도서관/놀이체험실 운영기관 페이지
- 한국관광공사/지자체 관광 정보

이 유형은 `비용`, `예약 방법`, `연령 제한`, `운영 주체`, `휴관 정보`가 중요하다.

### 3. 박물관·과학관·체험관·공공 실내시설

우선 소스:

- 운영기관 공식 홈페이지
- 공공 예약 페이지
- 한국관광공사 계열 관광 정보

민간 지도는 위치/후기 참고용으로만 사용한다.

### 4. 혜택·지원금 정보

무조건 공식 소스를 먼저 본다.

- 정부 부처 공식 페이지
- 정부24
- 지자체 공식 페이지
- 고용노동부/보건복지부/국세청/국가법령정보센터

민간 블로그는 참고용으로만 본다.

## 데이터 정규화 필드

시설 카드에 최소 포함할 필드:

- `id`
- `name`
- `region`
- `subRegion`
- `category`
- `ageBands`
- `indoorOutdoor`
- `priceType`
- `reservationRequired`
- `parking`
- `sourceType`
- `sourceUrl`
- `verifiedAt`
- `lastObservedAt`

권장 필드:

- `feedingRoom`
- `strollerFriendly`
- `rainFriendly`
- `stayMinutes`
- `operatorType`
- `editorNote`

## 검증 상태 모델

모든 시설/혜택 데이터에는 아래 상태 중 하나를 둔다.

- `official_verified`
- `semi_verified`
- `needs_refresh`
- `discovery_only`

발행 가능 상태는 `official_verified`와 `semi_verified`만 허용한다.

## 리프레시 정책

### 상업형 시설

- 가격/운영시간: 30일 주기 확인
- 휴무/운영 종료: 발행 전 최종 확인

### 공공 시설

- 분기별 기본 점검
- 방학/명절/휴관 공지는 시즌별 재확인

### 혜택/지원금

- 기준 날짜 명시
- 제도 변경 시 즉시 갱신
- 최소 월 1회 점검

## 콘텐츠 발행 워크플로우

1. 후보 수집
2. 공식 링크 확인
3. 표준 필드 입력
4. 검증 상태 부여
5. 글/카드 발행
6. 재검수 일정 등록

## 운영 금지 규칙

- 후기 텍스트를 사실 정보처럼 단정하지 않는다.
- 가격, 운영시간, 연령 제한을 민간 후기만으로 쓰지 않는다.
- 공식 출처가 없는 혜택 정보는 발행하지 않는다.

## 추천 시작 방식

초기에는 크롤러부터 만들지 않는다.

먼저 아래 방식으로 시작한다.

1. 서울 공공형 시설 시드 10~15건 구축
2. 경기 남부 상업형 시설 시드 10~15건 구축
3. 상업형/공공형을 같은 스키마로 넣되, 검증 방식은 분리 운영
4. 공식 검증 링크를 필수 입력
5. 허브 UI를 먼저 붙여 흐름 검증
6. 이후 반자동 수집을 검토

### Phase A 공급 모델

Phase A는 아래 두 버킷으로 운영한다.

1. `서울 공공형`
   - 공식 소스 기반
   - stale risk 낮음
   - 초기 허브 신뢰도 확보용
2. `경기 남부 상업형`
   - 수동 검증 기반
   - stale risk 높음
   - 검증 가능한 소수 시드부터 시작

## 신뢰도 메모

이 문서는 공식 사이트와 공공 포털 위주로 계획한 것이다.  
다만 상업형 키즈카페의 전국 최신 단일 공공 소스 부재 여부는 현재 확인 범위 기반 추론이므로, 완전히 정확하지 않을 수 있다.

간단한 결론:

공공 시설은 공식 소스로, 상업형 키즈카페는 민간 발견 + 공식 검증 조합으로 가야 한다.



# 📄 대출 계산기 & 중도상환수수료 계산기 PRD

## 1. 대출 계산기 (Loan Calculator)

### 1.1 기능 목적

사용자가 입력한 대출 조건을 기반으로 아래 정보를 계산한다.

- 매월 상환금액

- 총 상환금액

- 총 이자금액


지원 상환 방식

- 원리금균등

- 원금균등

- 만기일시상환


---

# 1.2 입력 필드 정의

|필드|설명|타입|예시|
|---|---|---|---|
|대출금액|대출받은 총 금액|number|100000000|
|대출기간(년)|대출 기간 (년 단위)|number|5|
|대출기간(월)|추가 대출 기간 (월 단위)|number|6|
|대출금리|연 이자율|number (%)|3.5|
|상환방법|상환 방식|enum|원리금균등 / 원금균등 / 만기일시|

---

# 1.3 계산에 사용되는 공통 변수

```text
대출금액 = P
연이율 = r
월이율 = r / 12 / 100
총기간 = n (개월)

n = (대출기간_년 * 12) + 대출기간_월
```

---

# 1.4 상환 방식별 계산 공식

---

# 1.4.1 원리금균등 상환

매월 동일 금액을 상환하는 방식

### 계산 공식

```
월상환금 = P × ( i × (1+i)^n ) / ( (1+i)^n − 1 )

P = 대출금액
i = 월이율
n = 총 상환개월
```

### 계산 절차

1️⃣ 월이율 계산

```
월이율 = 연이율 / 12 / 100
```

2️⃣ 총 상환 개월 계산

```
총개월 = (년 × 12) + 월
```

3️⃣ 월 상환금 계산

```
월상환금 = P × ( i × (1+i)^n ) / ((1+i)^n − 1)
```

4️⃣ 총 상환금

```
총상환금 = 월상환금 × n
```

5️⃣ 총 이자

```
총이자 = 총상환금 − 대출금액
```

---

# 1.4.2 원금균등 상환

매월 **원금은 동일**, 이자는 점점 감소

### 계산 공식

```
월 원금 상환액 = P / n
```

월 이자

```
잔여원금 × 월이율
```

### 계산 절차

1️⃣ 매월 원금

```
월 원금 = P / n
```

2️⃣ 월 이자

```
월 이자 = 남은 원금 × 월이율
```

3️⃣ 월 납입금

```
월 납입금 = 월 원금 + 월 이자
```

4️⃣ 총 이자

```
총이자 = 모든 월 이자 합
```

5️⃣ 총 상환금

```
총상환금 = 원금 + 총이자
```

특징

- 초기 납입금이 큼

- 총 이자가 가장 적음


---

# 1.4.3 만기일시 상환

대출 기간 동안 **이자만 납부**하고  
만기에 **원금을 일시 상환**

### 계산 공식

월 이자

```
월이자 = P × 월이율
```

총 이자

```
총이자 = 월이자 × n
```

총 상환금

```
총상환금 = P + 총이자
```

---

|항목|설명|
|---|---|
|월 상환금|매월 납부 금액|
|총 상환금|대출기간 동안 총 납부 금액|
|총 이자|총 이자 금액|

# 2. 중도상환수수료 계산기 (Prepayment Fee Calculator)

---

# 2.1 기능 목적

대출 계약 기간 이전에 상환할 경우  
금융기관에 납부해야 하는 **중도상환수수료** 계산

---

# 2.2 입력 필드 정의

|필드|설명|타입|예시|
|---|---|---|---|
|상환금액|중도 상환하려는 금액|number|30000000|
|수수료율|금융기관 수수료율|number (%)|1.2|
|대출일자|대출 실행일|date|2024-01-01|
|상환일자|실제 상환일|date|2025-01-01|
|상환기일|계약 만기일|date|2027-01-01|
|면제기간|중도상환수수료 면제 기간|number (년)|3|

---

# 2.3 중도상환수수료 기본 공식

일반적인 금융권 계산 방식

```
중도상환수수료
= 상환금액 × 수수료율 × (잔존기간 / 전체기간)
```

---

# 2.4 계산 변수 정의

```text
상환금액 = A
수수료율 = f
잔존기간 = R
전체기간 = T
```

---

# 2.5 기간 계산

### 전체 대출 기간

```
전체기간 = 상환기일 - 대출일자
```

### 잔존기간

```
잔존기간 = 상환기일 - 상환일자
```

단위

```
일(day)
```

---

# 2.6 면제기간 처리

면제기간 이후에는 수수료 없음

```
면제기간 = N년
```

조건

```
상환일자 >= 대출일자 + 면제기간
→ 수수료 = 0
```

---

# 2.7 최종 계산 공식

```
중도상환수수료
= 상환금액 × (수수료율 / 100) × (잔존일수 / 전체일수)
```

---

# 2.8 계산 절차

1️⃣ 전체 대출 기간 계산

```
전체기간 = 만기일 - 대출일
```

2️⃣ 남은 기간 계산

```
잔존기간 = 만기일 - 상환일
```

3️⃣ 면제기간 확인

```
상환일 >= 대출일 + 면제기간
→ 수수료 0
```

4️⃣ 수수료 계산

```
수수료 = 상환금액 × 수수료율 × (잔존기간 / 전체기간)
```

---

# 2.9 결과 출력

|---|---|

|잔존기간|남은 대출 기간|
|적용 수수료율|실제 적용된 수수료|

# Conclusion

대출 계산기는 **3가지 상환 방식 공식**을 기반으로 월 상환금, 총 상환금, 총 이자를 계산한다.  
중도상환수수료 계산기는 **상환금액 × 수수료율 × 잔존기간비율** 공식을 사용하며 **면제기간 이후에는 수수료가 0**이 된다.

다음은 **PRD 문서의 6번 섹션을 정리한 통합 버전**이다.  
요청대로 **JavaScript 계산 유의사항 + 큰 금액 제한**을 포함하고, **BigNumber/Decimal 라이브러리 내용은 제외**했다.

# 6. JavaScript 수학 계산 시 유의사항

대출 계산기 및 중도상환수수료 계산기는 **금융 계산**이므로 JavaScript 수학 계산 시 아래 사항을 고려해야 한다.

JavaScript는 **IEEE 754 부동소수점 방식**을 사용하기 때문에 다음 문제가 발생할 수 있다.

- 소수점 계산 오차

- 반올림 문제

- 매우 큰 숫자 정밀도 문제

- 금리 계산 오류


따라서 계산 정확도를 위해 **반올림 처리, 금리 변환, 입력 제한, 예외 처리**를 반드시 적용해야 한다.

---

# 6.1 부동소수점 계산 오차

JavaScript는 **2진 부동소수점 계산 방식**을 사용한다.

예시

```javascript
0.1 + 0.2
// 결과
0.30000000000000004
```

이러한 문제는 금융 계산에서 **금액 오차**를 발생시킬 수 있다.

### 대응 방법

계산 결과는 반드시 **반올림 처리**한다.

```javascript
Math.round(value)
```

또는

```javascript
Number(value.toFixed(0))
```

대출 계산기의 경우 일반적으로 **원 단위 반올림**을 사용한다.

---

# 6.2 금융 계산 반올림 규칙

대출 계산 결과는 보통 **원 단위 처리**가 필요하다.

사용 가능한 반올림 방식

|함수|설명|
|---|---|
|Math.round()|반올림|
|Math.floor()|버림|
|Math.ceil()|올림|

일반적인 금융 계산기에서는 다음 방식을 사용한다.

```javascript
Math.round(value)
```

---

# 6.3 금리 계산 처리

금리는 **퍼센트(%) 값**으로 입력되므로 계산 시 반드시 **소수 형태로 변환**해야 한다.

예

```text
입력 금리 = 3.5%
```

계산용 금리

```javascript
rate = 3.5 / 100
```

월 금리

```javascript
monthlyRate = rate / 12
```

예시 코드

```javascript
const annualRate = 3.5
const monthlyRate = annualRate / 100 / 12
```

---

# 6.4 거듭제곱 계산

원리금균등 계산 공식에서는 **거듭제곱 계산**이 필요하다.

JavaScript에서는 다음 함수를 사용한다.

```javascript
Math.pow(a, b)
```

또는

```javascript
a ** b
```

예

```javascript
(1 + monthlyRate) ** months
```

---

# 6.5 금리 0% 예외 처리

원리금균등 상환 공식은 금리가 **0%일 경우 오류**가 발생한다.

공식

```
P × ( i × (1+i)^n ) / ((1+i)^n − 1)
```

여기서

```
i = 0
```

이면 분모가 0이 된다.

따라서 별도 예외 처리가 필요하다.

```javascript
if (rate === 0) {
  monthlyPayment = principal / months
}
```

---

# 6.6 날짜 계산 주의사항

중도상환수수료 계산에서는 **날짜 차이 계산**이 필요하다.

JavaScript `Date` 객체를 사용할 경우 다음 문제를 주의해야 한다.

- timezone 영향

- 날짜 계산 오류

- 월 길이 차이


따라서 날짜 계산 시 **일(day) 단위 기준**으로 계산하는 것이 안전하다.

예시

```javascript
const diff = (endDate - startDate) / (1000 * 60 * 60 * 24)
```

# 6.8 입력 금액 제한 (Large Number Limit)

JavaScript `Number` 타입은 다음 범위까지만 정확한 정수 표현이 가능하다.

```
2^53 - 1
= 9,007,199,254,740,991
```

이 값을 초과하면 **정수 정밀도가 깨질 수 있다.**

또한 현실적인 금융 서비스에서는 **비정상적인 입력 방지**를 위해 금액 제한을 두는 것이 일반적이다.

---

# 권장 입력 제한

|항목|최대값|이유|
|---|---|---|
|대출금액|1,000,000,000,000 (1조)|현실적인 금융 계산 범위|
|상환금액|1,000,000,000,000 (1조)|중도상환 계산 동일|
|금리|0 ~ 100%|입력 오류 방지|
|대출기간|0 ~ 50년|현실적인 대출 기간|

---

# Validation 규칙

입력 값이 제한을 초과하면 계산하지 않는다.

예시

```javascript
const MAX_AMOUNT = 1_000_000_000_000

if (loanAmount > MAX_AMOUNT) {
  throw new Error("대출금액은 1조원을 초과할 수 없습니다.")
}
```

---

# UI 입력 제한

프론트 입력에서도 제한 적용

```html

```

또는

```javascript
if (value > MAX_AMOUNT) {
```

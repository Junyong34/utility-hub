# 로또 통계 데이터 분석 명세

## 목적

이 문서는 로또 추천 엔진에서 사용하는 통계 분석 항목과 계산 기준을 정의합니다. 모든 지표는 **번호 추천의 재미 요소**를 위한 참고 데이터이며, 당첨 확률 향상을 보장하지 않습니다.

## 입력 데이터

- 소스: `lib/lotto/lotto_draws.json`
- 단위 데이터: 회차, 당첨번호 6개, 보너스번호, 추첨일
- 분석 범위:
  - 전체 통계: 전체 회차
  - 최근 통계: 최근 N회차(기본 100)

## 분석 지표

### 1. 번호별 출현 빈도

- 정의: 번호(1~45)가 당첨번호 6개에 등장한 횟수
- 산식:
  - `count(number)`
  - `rate(number) = count(number) / totalRounds * 100`
- 활용:
  - 고빈도/저빈도 전략
  - AI 가중치의 빈도 항목

### 2. 최근 출현 빈도

- 정의: 최근 N회차에서 번호별 등장 횟수
- 산식:
  - `recentCount(number, N)`
- 활용:
  - 핫넘버 전략
  - AI 가중치의 최근 빈도 항목

### 3. 연속 미출현 회차

- 정의: 최신 회차 기준, 마지막 출현 이후 지난 회차 수
- 산식:
  - `absence(number) = latestRound - lastAppearanceRound`
- 활용:
  - 콜드넘버 전략
  - 미출현 전략 보조 지표

### 4. 홀짝 비율 분포

- 정의: 한 게임(6개 번호) 내 홀수:짝수 비율 집계
- 예시: `3:3`, `4:2`, `2:4`
- 활용:
  - 균형 전략 검증
  - AI 후처리 유효성 검증

### 5. 구간 분포

- 구간 정의:
  - 1-10, 11-20, 21-30, 31-40, 41-45
- 정의: 각 구간에서 등장한 번호의 비율
- 활용:
  - 균형 전략
  - AI 가중치 균형 보정

### 6. 동반 출현(Co-occurrence)

- 정의: 번호 쌍 `(a, b)`가 같은 회차에 함께 등장한 횟수
- 산식:
  - `coCount(a, b)`
  - 상위 N개 쌍 추출(기본 20)
- 활용:
  - AI 가중치의 동반 출현 항목

### 7. 번호 합계 분포

- 정의: 한 게임의 번호 6개 합계
- 산식:
  - `sum = n1 + n2 + ... + n6`
- 활용:
  - 품질 점검(권장 범위 참고)

## 검증 규칙

추천 결과 번호에 대해 다음 규칙을 적용할 수 있습니다.

- 기본 규칙:
  - 6개 번호
  - 1~45 범위
  - 중복 없음
- 선택 규칙:
  - 홀짝 균형(권장 2:4 ~ 4:2)
  - 구간 과밀 방지(단일 구간 4개 이상 지양)
  - 연속번호 과밀 방지(3개 이상 연속 지양)

## 구현 파일 매핑

- 통계 분석 엔진: `lib/lotto/algorithms/statistics-analyzer.ts`
- AI 추천 엔진: `lib/lotto/algorithms/ai-statistical-recommendation.ts`
- 확률통계 전략: `lib/lotto/algorithms/probability-statistical-recommendation.ts`
- 유효성 검증: `lib/lotto/algorithms/validation.ts`

## 주의사항

- 본 분석은 과거 데이터 기반의 참고 정보입니다.
- 로또 추첨은 무작위이므로 모든 번호 조합의 당첨 확률은 동일합니다.

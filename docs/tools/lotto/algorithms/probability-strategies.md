# 확률통계 기반 추천 전략 (6가지)

## 개요

본 문서는 로또 번호 추천 서비스에서 제공하는 **6가지 확률통계 전략**의 상세 알고리즘과 사용 시나리오를 정의합니다.

PRD 6.4 명세에 따라, 사용자는 자신의 선호에 따라 전략을 직접 선택할 수 있습니다.

## 전략 목록

| 전략 ID | 전략명 | 영문 | 설명 |
|--------|--------|------|------|
| 1 | 고빈도 전략 | High Frequency | 전체 회차에서 가장 자주 출현한 번호 |
| 2 | 저빈도 전략 | Low Frequency | 전체 회차에서 출현 빈도가 낮은 번호 |
| 3 | 미출현 전략 | Undrawn | 최근 N회차 내에 나오지 않은 번호 |
| 4 | 균형 전략 | Balanced | 홀짝 3:3, 각 구간 균형 |
| 5 | 핫넘버 전략 | Hot Numbers | 최근 10회차에서 자주 출현 |
| 6 | 콜드넘버 전략 | Cold Numbers | 30회차 이상 미출현 |

---

## 1. 고빈도 전략 (High Frequency)

### 개념
**"과거에 자주 나온 번호가 앞으로도 나올 가능성이 있다"**는 가정 하에, 전체 회차 출현 빈도 상위 번호를 선택합니다.

### 알고리즘

```
[1] 전체 회차(1~1,000회) 데이터에서 번호별 출현 빈도 계산
[2] 출현 횟수 기준 내림차순 정렬
[3] 상위 15개 번호를 "고빈도 풀"로 설정
[4] 고빈도 풀에서 무작위로 6개 선택
[5] 오름차순 정렬 후 반환
```

### 예시

| 번호 | 출현 횟수 | 출현율 |
|------|----------|--------|
| 34 | 120 | 12.0% |
| 27 | 115 | 11.5% |
| 13 | 112 | 11.2% |
| 45 | 110 | 11.0% |
| 17 | 108 | 10.8% |
| ... | ... | ... |

**선택 예시**: [13, 17, 27, 34, 38, 45]

### 사용 시나리오
- 통계를 신뢰하는 사용자
- 안정적인 번호를 원하는 경우
- "자주 나온 번호는 이유가 있다"고 생각하는 사용자

---

## 2. 저빈도 전략 (Low Frequency)

### 개념
**"오랫동안 안 나온 번호가 이제 나올 차례"**라는 도박사의 오류(Gambler's Fallacy)를 재미 요소로 활용합니다.

### 알고리즘

```
[1] 전체 회차 데이터에서 번호별 출현 빈도 계산
[2] 출현 횟수 기준 오름차순 정렬
[3] 하위 15개 번호를 "저빈도 풀"로 설정
[4] 저빈도 풀에서 무작위로 6개 선택
[5] 오름차순 정렬 후 반환
```

### 예시

| 번호 | 출현 횟수 | 출현율 |
|------|----------|--------|
| 2 | 78 | 7.8% |
| 41 | 82 | 8.2% |
| 39 | 85 | 8.5% |
| 6 | 87 | 8.7% |
| 11 | 89 | 8.9% |
| ... | ... | ... |

**선택 예시**: [2, 6, 11, 29, 39, 41]

### 사용 시나리오
- 역발상을 선호하는 사용자
- 인기 없는 번호로 배당률↑ 기대 (실제로는 배당률 동일)
- 재미 요소 추구

---

## 3. 미출현 전략 (Undrawn)

### 개념
**최근 N회차(기본 20회) 동안 한 번도 나오지 않은 번호**를 선택합니다.

### 알고리즘

```
[1] 최근 20회차의 당첨번호를 집합으로 저장
[2] 1~45 중 집합에 없는 번호를 "미출현 풀"로 설정
[3] 미출현 풀이 6개 미만이면:
    → 저빈도 전략 번호로 보충
[4] 미출현 풀에서 무작위로 6개 선택
[5] 오름차순 정렬 후 반환
```

### 파라미터
- `recentWindow`: 최근 회차 윈도우 (기본 20회)

### 예시

**최근 20회차 출현 번호**: [1, 2, 3, 5, 7, 8, 10, 12, 13, ...]
**미출현 번호**: [4, 6, 9, 11, 15, 18, 19, 22, ...]

**선택 예시**: [4, 9, 15, 19, 28, 37]

### 사용 시나리오
- 트렌드 변화를 기대하는 사용자
- "최근 안 나온 번호"에 주목하는 전략
- 단기 패턴 회피

---

## 4. 균형 전략 (Balanced)

### 개념
**홀짝 비율 3:3, 각 구간(1~10, 11~20, ...)에서 최소 1개씩** 선택하여 분산을 극대화합니다.

### 알고리즘

```
[1] 5개 구간을 정의:
    - 1~10, 11~20, 21~30, 31~40, 41~45
[2] 각 구간에서 랜덤으로 1개씩 선택 (총 5개)
[3] 6번째 번호를 전체 범위에서 랜덤 선택
[4] 홀짝 비율 검증:
    - 홀수 3개, 짝수 3개가 아니면 재생성
[5] 최대 50회 재시도 후 반환
```

### 예시

| 구간 | 선택 번호 |
|------|----------|
| 1~10 | 7 (홀수) |
| 11~20 | 14 (짝수) |
| 21~30 | 23 (홀수) |
| 31~40 | 36 (짝수) |
| 41~45 | 42 (짝수) |
| 추가 | 15 (홀수) |

**선택 결과**: [7, 14, 15, 23, 36, 42] ✓ (홀3:짝3)

### 사용 시나리오
- 구간 편중을 피하고 싶은 사용자
- 통계적 평균에 가까운 번호 선호
- 균형잡힌 조합 선호

---

## 5. 핫넘버 전략 (Hot Numbers)

### 개념
**최근 10회차 중 2회 이상 출현한 "핫한 번호"**를 선택합니다.

### 알고리즘

```
[1] 최근 10회차의 당첨번호 수집
[2] 번호별 출현 빈도 집계
[3] 출현 횟수 >= 2인 번호를 "핫 풀"로 설정
[4] 핫 풀이 6개 미만이면:
    → 고빈도 전략 번호로 보충
[5] 핫 풀에서 무작위로 6개 선택
[6] 오름차순 정렬 후 반환
```

### 파라미터
- `hotThreshold`: 핫 판정 기준 (기본 2회)

### 예시

**최근 10회차 출현 빈도**:

| 번호 | 출현 횟수 |
|------|----------|
| 13 | 3회 ⭐ |
| 27 | 3회 ⭐ |
| 34 | 2회 ⭐ |
| 7 | 2회 ⭐ |
| 19 | 2회 ⭐ |
| ... | 1회 이하 |

**선택 예시**: [7, 13, 19, 27, 34, 38]

### 사용 시나리오
- 최근 추세를 따르는 모멘텀 전략
- "요즘 자주 나오는 번호" 선호
- 단기 트렌드 추종

---

## 6. 콜드넘버 전략 (Cold Numbers)

### 개념
**30회차 이상 연속으로 나오지 않은 "얼어붙은 번호"**를 선택합니다.

### 알고리즘

```
[1] 모든 번호의 마지막 출현 회차를 조회
[2] 현재 회차 - 마지막 출현 회차 >= 30인 번호를 "콜드 풀"로 설정
[3] 콜드 풀이 6개 미만이면:
    → 저빈도 전략 번호로 보충
[4] 콜드 풀에서 무작위로 6개 선택
[5] 오름차순 정렬 후 반환
```

### 파라미터
- `coldThreshold`: 콜드 판정 기준 (기본 30회차)

### 예시

**최근 1,000회차 기준**:

| 번호 | 마지막 출현 | 연속 미출현 |
|------|------------|------------|
| 2 | 970회 | 30회 ⭐ |
| 18 | 965회 | 35회 ⭐ |
| 29 | 958회 | 42회 ⭐ |
| 41 | 952회 | 48회 ⭐ |
| ... | ... | ... |

**선택 예시**: [2, 18, 29, 35, 41, 44]

### 사용 시나리오
- "이제 나올 때가 됐다" 기대
- 미출현 전략의 강화 버전
- 역발상 극단 전략

## 통계적 사실

### ⚠️ 중요 공지

**모든 전략의 실제 당첨 확률은 동일합니다 (약 1/8,145,060).**

- 로또는 완전 무작위 추첨이므로, 과거 데이터는 미래 결과에 영향을 주지 않습니다.
- 고빈도 번호와 저빈도 번호의 당첨 확률은 수학적으로 동일합니다.
- 이 전략들은 **사용자에게 선택의 재미와 근거를 제공하기 위한 엔터테인먼트 요소**입니다.

### 도박사의 오류 (Gambler's Fallacy)

> "오랫동안 안 나온 번호가 곧 나올 것이다"는 믿음은 **도박사의 오류**입니다.
>
> 각 추첨은 독립 사건이며, 이전 결과는 다음 결과에 영향을 주지 않습니다.

### 서비스 철학

본 서비스는:
- 사용자에게 투명하게 알고리즘을 공개합니다.
- 당첨을 보장하거나 확률 향상을 주장하지 않습니다.
- 책임감 있는 구매를 권장합니다.

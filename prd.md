# 수익형 블로그 + 유틸 웹사이트 프로젝트 계획

## 1. 프로젝트 목표

### 1차 목표
- SSG + SSR 기반 웹사이트 MVP 오픈
- 실제 서버 배포 완료
- 기본 SEO 세팅 완료
- 광고 삽입 구조 설계

※ 수익은 목표가 아니라 결과로 둔다.

---

## 2. 프로젝트 방향

### 전략
- 주제는 우선 확정하지 않음
- 플랫폼(엔진) 먼저 구축
- 이후 콘텐츠와 유틸을 확장

### 사이트 구조
- SSG: 블로그 포스트
- SSR: 계산기 / 유틸 페이지
- 내부 링크 기반 트래픽 구조 설계
- SEO 자동화

---

## 3. 기술 설계 (MVP 기준)

### 프레임워크
- Next.js (App Router 권장)

### 콘텐츠
- Markdown 또는 MDX 기반 포스트 관리
- 정적 생성(SSG) 적용

### 유틸 페이지
- 서버 렌더링(SSR) 기반
- 계산기 / 변환기 형태

### 필수 요소
- 메타태그 자동 생성
- sitemap 자동 생성
- robots.txt 구성
- 반응형 디자인
- 광고 삽입 위치 설계

---

## 4. 2주 실행 계획

### Week 1
- 기본 레이아웃 구성
- 포스트 리스트 페이지
- 포스트 상세 페이지 (SSG)
- 유틸 페이지 1개 구현
- SEO 메타 기본 구성

### Week 2
- sitemap / robots 설정
- 광고 영역 설계
- Search Console 등록
- 프로덕션 배포

---

## 5. 출시 후 확장 계획

### Phase 1
- 글 10~20개 작성
- 유틸 2~3개 추가
- 내부 링크 최적화

### Phase 2
- 검색 유입 데이터 분석
- 상위 노출 글 리라이팅
- CTR 개선

### Phase 3
- 수익 구조 최적화
- 고단가 키워드 확장
- 특정 카테고리 집중 전략

---

## 6. 성공 기준

- 실제 배포된 URL 존재
- Google Search Console 등록 완료
- 포스트 최소 10개 이상
- 유틸 최소 2개 이상
- 광고 심사 통과

---

## 7. 핵심 원칙

- 완벽한 설계보다 빠른 출시
- 기능 과잉 금지
- 콘텐츠 중심 구조
- 문제 해결형 사이트 지향
- 6개월 단위 장기 관점 유지
